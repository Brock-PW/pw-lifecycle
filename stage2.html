Lead Type Decision

Brock
Two client types: Standard PW Contractor and Sub Oversight. Mid-project rescue is a condition that applies to either — not a separate type.

Standard PW Contractor

Brock / Anthy
Most common. Contractor performing public works. Needs CPR production, DAS forms, rate calculation, ECPR upload, CAC invoices. Mid-project rescue applies if behind on CPRs or received DIR notice.

Sub Oversight

Brock / Anthy
Prime contractor managing subs. Joint liability exposure. Call may include CEO, CFO, HR, accountants. Same close process, different estimate. Mid-project rescue applies if subs are non-compliant.

⚑ Mid-Project Rescue — Standard PW

Brock / Anthy
Contractor is already on a project and is behind, missed CPRs, or received DIR notice. Estimate includes restitution calculation ($35/employee/week) and catch-up CPR production. Higher urgency — faster close.

⚑ Mid-Project Rescue — Sub Oversight

Brock / Anthy
Subs are non-compliant. Prime at risk of joint liability. Estimate includes sub restitution review and audit risk assessment. Higher urgency — faster close.

STAGE 2 SOP
📞 Consultation Call
Goal
Understand the prospect's situation, gather everything needed to build their estimate, set clear expectations on what comes next, and confirm a follow-up window.
Pre-Call — What Claude Surfaces

Whatever is known from Stage 1: how they found us, any emails or messages, company name, project mentions, prior compliance history. Sometimes this is a full brief. Sometimes it is almost nothing. Use what exists — don't manufacture what doesn't.

Calendar & recording rule: Every consultation call must be recorded in Otter.ai. The Otter recording is the trigger for all post-call automation. No Otter recording = no automation fires.

Calendar invite naming: Must contain the lead name (e.g. "Jane Doe — Larry's Construction"). This is how Claude connects the transcript to the correct Zoho record.
On the Call — What to Gather
• Location
• DIR number (if they have one)
• Project duration
• Sub or oversight client
• Active employee count
• Number of contractors (if oversight)
• Decision maker
• Who receives emails
• Expected response window
• Best time to follow up
Roles
🤖 CLAUDE DOES
  • • Prepares pre-call brief
  • • Extracts Otter transcript post-call
  • • Creates Notion client brain page
  • • Drafts follow-up email + PDF estimate
  • • Surfaces follow-ups on schedule
  • • Resets sequence on any response
  • • Updates Zoho stage + tasks
  • • Drafts Stage 3 welcome email
👤 HUMAN DOES
  • • Runs the call + builds rapport
  • • Listens for compliance gaps
  • • Gathers all estimate inputs
  • • Confirms follow-up window
  • • Reviews + approves each email
  • • Answers prospect questions
Human Actions — In Order
  1. Review pre-call brief — Claude surfaces this before the call. Read it even if it's thin.
  2. Run the call and gather all inputs — Confirm follow-up window before hanging up.
  3. Confirm call complete in dashboard — This triggers all post-call automation. Nothing fires until confirmed.
  4. Review and approve Day 0 package — Email, PDF estimate, contract, PW overview. Send same day.
  5. Approve each follow-up as it surfaces — Days 3, 8, 13, 27. Nothing sends without approval.
  6. Answer prospect questions + mark answered — Any response resets the sequence to Day 0.
  7. Execute and return signed contract (Brock) — Counter-sign, approve Stage 3 welcome email. Closes Stage 2, opens Stage 3.
Follow-Up Sequence
Day 0
Full package sent same day — personalized email + PDF estimate + contract + PW overview
Day 3
First follow-up — draws from Otter transcript and all Notion context since Day 0
Day 8
Second follow-up — updated with any new context, replies, or conversations logged to Notion
Day 13
Third follow-up — same contextual approach, angled toward contract signature
Day 27
Final touch — genuine, low-pressure close of the loop
Reset rule: Any prospect response resets sequence to Day 0. Claude drafts new email from all accumulated context.
Stop rule: Sequence stops permanently when contract is signed.
No response: After Day 27 → moves to Stage 5 retention queue.
What Success Looks Like
✓Prospect understands what PW does
✓All estimate inputs collected on the call
✓Follow-up window confirmed before hanging up
✓Day 0 package sent same day
Watch For
Active project needing restitution — treat as urgent, compress the timeline
Bid estimate only — lower urgency, adjust cadence expectations
Large organization — escalate, may require special handling or Brock involvement
